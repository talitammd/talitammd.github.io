Document Object Model

基本概念

DOM，即文档对象模型。它是一种跨平台的、独立于编程语言的API，它把HTML、XHTML或XML文档当作一个树结构，而每个节点视为一个对象，这些对象可以被编程语言操作，进而改变文档的结构，映射到文档的显示。

简单来说，DOM就是我们为了方便编程语言对HTML等文档进行操作，把HTML文档中的节点全部视为一个个的对象，所有可用于操作和创建网页的属性、方法和事件都被组织到对象中，这些对象依照层级关系形成一棵树，这棵树就命名为DOM树。有了对象，编程就方便多了，只要一层层拿到对象就可以优雅地改变对象的属性进而动态地改变HTML等文档的展示。

例如，DOM 中指定下面代码中的 querySelectorAll 方法必须要返回文档中所有 <p> 元素组成的列表：

const paragraphs = document.querySelectorAll("p");
// paragraphs[0] 是第一个 <p> 元素
// paragraphs[1] 是第二个 <p> 元素，依此类推
alert(paragraphs[0].nodeName);

DOM 不是一种编程语言，也不是 JavaScript 语言的一部分，而是用于建立网站的 Web API。DOM 被设计成与特定编程语言相独立，使文档的结构化表述可以通过单一，一致的 API 获得。

HTML文档

在DOM树中，HTML文档中的每个元素都被表示为一个节点。这些节点具有属性和方法，允许你通过编程的方式与它们进行交互。

当HTML文档被加载到浏览器中时，浏览器会解析HTML并创建一个DOM树，这个DOM树代表了整个HTML文档的结构。DOM树中的每个节点都对应着HTML文档中的一个元素或属性。

以下是HTML中常见的元素节点类型及其分类列举：

  1. 文档结构元素：

    • <html>: 根元素，包含了整个HTML文档。
    • <head>: 包含了文档的元（meta）数据，如<title>, <style>, <meta>, <link>, <script>等。
    • <body>: 包含了所有可见的页面内容，如文本、图片、音频、视频等。

  2. 标题元素：

    • <h1>, <h2>, <h3>, <h4>, <h5>, <h6>: 定义标题或子标题，<h1>表示最高级别的标题。

  3. 段落元素：

    • <p>: 定义段落。

  4. 列表元素：

    • <ul>: 无序列表（unordered list）。
    • <ol>: 有序列表（ordered list）。
    • <li>: 列表项（list item）。

  5. 链接元素：

    • <a>: 定义超链接，可以链接到其他网页或同一网页的不同部分

      • href：指定链接的目标地址
      • target：指定在何处打开链接的文档
      • rel：定义当前文档与被链接文档之间的关系
      <a href="https://example.com" 
target="_blank" 
rel="noopener noreferrer">链接文本</a>

  6. 图像元素：

    • <img>: 定义图像，通常与src、alt等属性一起使用

      • src：指定图像的来源
      • alt：为图像提供替代文本，以便在图像无法显示时显示
      • width 和 height：指定图像的宽度和高度
      <img src="image.jpg" alt="示例图片" width="500" height="600">

  7. 表格元素：

    • <table>: 定义表格。
    • <tr>: 定义表格行。
    • <th>: 定义表格头部单元格。
    • <td>: 定义表格数据单元格。

  8. 表单元素：

    • <form>: 定义HTML表单，用于用户输入。
    • <input>: 定义输入字段，如文本框、密码框、单选框、复选框等
      • type、name、value
    • <textarea>: 定义多行文本输入字段
      • rows、cols
    • <button>: 定义点击按钮。
    • <select>: 定义下拉选择列表
      • name、value
    • <option>: 定义下拉选择列表中的选项
      • name、value
    <input type="text" name="username" value="默认值">  
<textarea rows="4" cols="50">默认文本</textarea>  
<select name="color">  
  <option value="red">红色</option>  
  <option value="blue">蓝色</option>  
</select>

  9. 分区和布局元素：

    • <div>: 定义一个区块或容器，通常与CSS一起使用来布局页面。
    • <span>: 定义内联元素，通常用于对文本的一部分应用样式。
    • <section>: 定义文档中的独立部分或章节。
    • <article>: 定义独立的内容，如博客文章、论坛帖子等。
    • <nav>: 定义导航链接的容器。
    • <header>: 定义文档的页眉或区域页眉。
    • <footer>: 定义文档的页脚或区域页脚。

  10. 媒体元素：

    • <audio>: 定义音频内容。
    • <video>: 定义视频内容。

  11. 其他常见元素：

    • <br>: 插入一个简单的换行符。
    • <hr>: 创建水平线。
    • <meta>: 提供关于HTML文档的元信息。
    • <link>: 定义文档与外部资源之间的关系。
    • <style>: 定义内部样式信息。
    • <script>: 定义客户端脚本，如JavaScript代码。

网页加载过程

  1. HTML下载到本地内存：当浏览器请求一个HTML页面时，会从服务器下载HTML文件并将其存储在本地内存中。这个过程是通过HTTP请求完成的，并且通常会遵循TCP/IP协议
  2. 解析成DOM树：浏览器会解析HTML文档，将HTML标签和属性转换为DOM树
  3. 展示页面到屏幕：在构建DOM树的同时或之后，浏览器还会开始解析CSS并构建CSSOM（CSS对象模型）树。然后，它会将DOM树和CSSOM树合并成一个渲染树（render tree），这个渲染树包含了页面上需要显示的内容以及这些内容如何显示（例如颜色、字体大小等）。最后，浏览器会根据渲染树在屏幕上绘制页面
  4. DOM API 动态修改：DOM API允许JavaScript等网络编程语言动态地访问和更新HTML文档的内容、结构和样式。例如，你可以使用JavaScript来修改DOM树中的元素、添加或删除元素、更改元素的属性或样式等
    • 在页面加载过程中：当浏览器接收到 HTML 文档并开始解析时，它会同时或随后构建一个 DOM 树。一旦 DOM 树构建完成，浏览器就可以开始渲染页面了，但此时页面可能还没有完全加载完成，比如可能还有 CSS 和 JavaScript 文件需要加载和执行。此时，JavaScript 可以使用 DOM API 来修改已经解析并添加到 DOM 树中的元素
    • 在页面加载完成后：一旦页面完全加载并渲染完成（例如，所有图片都已加载），JavaScript 可以使用 DOM API 来添加新元素、删除现有元素或执行其他操作。这通常通过监听 DOMContentLoaded 或 load 事件来实现，这些事件会在页面加载的不同阶段触发
  5. 网页加载优化：为了提高网页的加载性能，开发者可以采取一系列优化措施，如压缩HTML、CSS和JavaScript文件、使用CDN（内容分发网络）来加速资源的加载、通过异步加载和延迟加载来减少阻塞等

事件对象

简单来讲，事件就是一个动作，而 HTML 为这些动作指定了特定的对象，和 DOM 对象一样，事件不属于任何一门语言，但是可以作为元素（DOM对象）的属性嵌入元素中，也就是指定该元素为触发对象，当在该元素中触发了指定的事件时，就可以执行该事件绑定在该元素上的JavaScript代码。

DOM中的事件是如何触发的？

  1. 事件源：事件的触发首先需要一个事件源，也就是引发事件的HTML元素。例如，当用户点击一个按钮时，这个按钮就是事件源。
  2. 事件类型：每个事件都有一个类型，如“click”、“keydown”等。当事件源上发生与事件类型匹配的动作时，事件就会被触发。
  3. 事件监听器：为了处理事件，需要在事件源上注册一个或多个事件监听器（也称为事件处理器）。事件监听器是一个函数，当事件被触发时，这个函数会被调用。在JavaScript中，通常使用addEventListener()方法为元素添加事件监听器。
  4. 事件流：DOM事件流描述了事件从发生到被处理的过程。这个过程包括三个阶段：捕获阶段、目标阶段和冒泡阶段。
    • 捕获阶段：从document对象开始，沿着DOM树向下传播，直到到达事件源。在捕获阶段，事件监听器可以被注册在DOM树的任何节点上，但默认情况下，事件监听器不会在捕获阶段被触发（除非在注册时特别指定）。
    • 目标阶段：当事件到达事件源时，会触发注册在该元素上的事件监听器。
    • 冒泡阶段：事件从事件源开始，沿着DOM树向上传播，直到回到document对象。在冒泡阶段，所有经过的节点（包括事件源本身）上注册的事件监听器都会被触发。举个例子，两个元素A和B绑定了不同的点击事件，并且B元素在A元素的内部，那么在点击了B时，B事件会先发生，然后A事件发生。
  5. 事件处理：当事件监听器被触发时，会执行相应的函数。这个函数可以执行任何需要的操作，如修改页面内容、发送网络请求等。

注意：

  1. 不同浏览器对DOM事件的支持可能有所不同。例如，一些较旧的浏览器可能不支持某些事件类型或事件流特性
  2. DOM 0级事件（直接在HTML元素上添加事件监听器，如element.onclick = function() {...};）和DOM 2级事件（使用addEventListener()方法添加事件监听器）在事件处理机制上也有所不同。DOM 0级事件只能在事件的目标阶段处理事件，而DOM 2级事件则可以在捕获阶段、目标阶段和冒泡阶段处理事件。

以下是一些常见的DOM事件：

  1. 鼠标事件：
    • click：当用户在元素上单击鼠标按钮时触发。
    • dblclick：当用户在元素上双击鼠标按钮时触发。
    • mousedown：当用户在元素上按下鼠标按钮时触发。
    • mouseup：当用户在元素上释放鼠标按钮时触发。
    • mousemove：当鼠标指针在元素内部移动时触发。
    • mouseover：当鼠标指针从元素外部移动到元素内部时触发。
    • mouseout：当鼠标指针从元素内部移动到元素外部时触发。
    • contextmenu：当用户在元素上右键点击时触发（通常用于显示上下文菜单）。
  2. 键盘事件：
    • keydown：当用户在键盘上按下某个键时触发。
    • keyup：当用户在键盘上释放某个键时触发。
    • keypress：当用户在键盘上按下并释放一个键时触发（通常用于处理文本输入）。
  3. 表单事件：
    • submit：当表单提交时触发（常用于验证表单数据）。
    • reset：当表单重置时触发。
    • focus：当元素获得焦点时触发（例如，用户点击输入框）。
    • blur：当元素失去焦点时触发（例如，用户离开输入框）。
    • change：当元素的值发生变化且失去焦点时触发（常用于输入框和下拉列表）。
  4. 文档/窗口事件：
    • load：当页面或图片加载完成时触发。
    • unload：当用户离开页面时触发（例如，关闭页面或跳转到其他页面）。
    • scroll：当用户滚动页面时触发。
    • resize：当浏览器窗口大小发生变化时触发。
  5. 触摸事件（主要针对移动设备）：
    • touchstart：当用户触摸屏幕时触发。
    • touchmove：当用户在屏幕上移动手指时触发。
    • touchend：当用户从屏幕上移开手指时触发。
    • touchcancel：当系统取消触摸事件时触发（例如，电话呼入）。
  6. 拖放事件：
    • dragstart：当用户开始拖动元素时触发。
    • drag：当用户在拖动元素时触发。
    • dragend：当用户结束拖动元素时触发。
    • dragenter：当拖动的元素进入目标元素时触发。
    • dragover：当拖动的元素在目标元素上移动时触发。
    • dragleave：当拖动的元素离开目标元素时触发。
    • drop：当用户在目标元素上释放拖动的元素时触发。
  7. 其他事件：
    • error：当发生错误时触发（例如，图片加载失败）。
    • abort：当加载被中止时触发（例如，用户停止加载页面）。
    • beforeunload：当页面即将卸载（关闭、刷新或跳转到其他页面）时触发。

Browser Object Model

BOM是浏览器对象模型，DOM 是将HTML文档中的各个元素封装成一个对象（Document），而 BOM 则是将一个浏览器的各个组成部分封装成对象（Browser）供调用使用。

BOM的核心对象是window，它表示浏览器的一个实例（通常就是浏览器窗口或标签页）。由于window是BOM的顶层对象，其他BOM对象都是它的子对象或者方法。

BOM的组成部分

图中看出，BOM 对象大致包含五个部分：

  • Window 对象：浏览器窗口对象
  • Navigator：浏览器对象
  • Screen：浏览器所处客户端的显示器屏幕对象
  • History：浏览器当前窗口的访问历史记录对象
  • Location：浏览器当前窗口的地址栏对象

窗口对象window

BOM 的核心是 window 窗口对象对象，表示浏览器的实例。window 对象在浏览器中有两重身份，一个是全局对象，另一个就是浏览器窗口的 JavaScript 接口。这意味着网页中定义的所有对象、变量和函数都以 window 作为其 Global 对象，都可以访问其上定义全局方法。

window对象不需要创建可以直接使用，window.方法名();，甚至可以把window直接省略方法名();

window.alert("hello window");
//两种写法都可以
alert("hello window");

以下是一些常见的Window对象方法：

  1. alert()：显示一个带有指定消息和OK按钮的警告框

  2. confirm()：显示一个带有指定消息以及确定和取消按钮的对话框。如果用户点击确定按钮，则返回true；如果用户点击取消按钮或关闭对话框，则返回false

    var userInput = window.confirm("你确定要继续吗?");  
if (userInput) {  
  console.log("用户点击确定");  
} else {  
  console.log("用户点击取消或关闭");  
}

  3. prompt()：显示一个对话框，提示用户输入一些文本。对话框包含一个确定按钮和一个取消按钮。如果用户点击确定按钮，则返回用户输入的文本；如果用户点击取消按钮或关闭对话框，则返回null

    var userName = window.prompt("请输入你的姓名:", "默认姓名");  
if (userName != null) {  
  console.log("你好，" + userName + "!");  
} else {  
  console.log("用户未输入姓名或点击了取消");  
}

  4. open()：打开一个新的浏览器窗口或查找一个已命名的窗口。此方法接收 4个参数：要加载的 URL、目标窗口、特性字符串和表示新窗口在浏览器历史记录中是否替代当前加载页面的布尔值

    window.open("https://www.example.com", 
"_blank", 
"width=500,height=500");

  5. close()：关闭当前窗口或指定的窗口。但是，需要注意的是，此方法只对通过JavaScript打开的窗口有效，对于用户直接打开的窗口（如浏览器中的标签页）则不起作用

    <button id="openWindow">打开窗口</button>
<button id="closeWindow">关闭窗口</button>
<script>
    var openWindow = document.getElementById("openWindow");
    var newWindow; //全局变量，对新创建的窗口进行操作
    openWindow.onclick = function(){
        newWindow = open("https://www.baidu.com"); //打开指定url的窗口
    }
        
    var closeWindow = document.getElementById("closeWindow");
    closeWindow.onclick = function(){
        newWindow.close(); //关闭调用它的窗口
    }
</script>

  6. setTimeout() 和 clearTimeout()：用于在指定的毫秒数后调用函数或计算表达式。clearTimeout() 方法用于取消由 setTimeout() 方法设置的 timeout

    var myTimer = setTimeout(function() {  
  console.log("Hello, World!");  
}, 2000); // 2秒后执行  
  
// 取消定时器  
clearTimeout(myTimer);

  7. setInterval() 和 clearInterval()：与 setTimeout() 类似，但 setInterval() 是在每个指定的间隔重复执行指定的函数或代码。clearInterval() 方法用于取消由 setInterval() 方法设置的间隔

  8. resizeTo()：调整窗口大小，有两个参数，分别设置窗口的高度和宽度。

  9. moveTo()：移动窗口，两个参数，接收的是新位置的x和y坐标值。

window对象中的属性

  • 第一类：window中的属性包括其他对象

    • history：该对象保存着用户上网的历史记录。出于安全方面的考虑，开发人员无法得知用户浏览过的URL，不过借由用户访问过的页面列表，同样可以在不知道实际URL的情况下实现后退前进

      • length：返回历史列表中的网址数
      • back()：加载 history 列表中的前一个 URL
      • forward()：加载 history 列表中的下一个 URL
      • go()：加载 history 列表中的某个具体页面，负数表示向后跳转，正数表示向前跳转

    • location：浏览器当前窗口的地址栏对象，通过window.location对象，可以解析URL、重定向浏览器到新的页面、重新加载当前页面等

      • window.location.href：获取或设置当前页面的完整URL。
      • window.location.protocol：获取URL的协议部分（如 "http:" 或 "https:"）。
      • window.location.host：获取URL的主机名和端口号（如果有）。
      • window.location.hostname：获取URL的主机名。
      • window.location.pathname：获取URL的路径名。
      • window.location.search：获取URL的查询字符串部分（包括问号）。
      • window.location.hash：获取URL的哈希（锚点）部分。

      location对象中的方法：

<button>替换页面</button>
<button>刷新页面</button>
<script>
  var btn = document.getElementsByTagName('button');//返回值为一个类数组
  btn[0].onclick = function () {
    location.assign('./test2.html')
  }
  btn[1].onclick = function () {
    location.replace('./test2.html')
  }
  btn[2].onclick = function () {
    location.reload()
  }
</script>
      • assign()：传递一个url参数，打开新url，替换当前页面，并在浏览记录中生成一条记录，页面替换后可以返回上一个页面
      • replace()：参数为一个url，打开新url，替换当前页面，不会在历史记录中生成新记录，替换页面后无法再次返回之前页面
      • reload()：重新加载当前显示的页面，参数可以为boolean类型，默认为false，表示以最有效方式重新加载，可能从缓存中直接加载。如果参数为true，强制从服务器中重新加载

    • navigator：浏览器对象，提供了与浏览器相关的信息和操作接口。在前端开发中，通常使用Navigator对象来获取用户浏览器的相关信息，例如浏览器的名称、版本、语言设置、设备类型、操作系统等。这些信息对于实现浏览器兼容性、设备适配等功能非常重要。Navigator对象还包含了一些方法和属性，用于获取当前设备的地理位置信息、访问设备的媒体设备等

      • navigator.appName：浏览器名称；注意：“Netscape”是 IE11、Chrome、Firefox 以及 Safari 的应用程序名称的统称；
      • navigator.appVersion：浏览器版本；
      • navigator.appCodeName：返回浏览器的应用程序代码名称
      • navigator.language：浏览器设置的语言；
      • navigator.platform：操作系统类型；
      • navigator.userAgent：返回用户代理头的字符串表示（通常是浏览器类型、版本和操作系统信息）；
      • navigator.cookieEnabled：返回一个布尔值，表示是否启用cookie；
      • navigator.product：返回浏览器引擎的产品名称

    • screen

      • screen.width和screen.height：屏幕总宽度/高度
      • screen.availWidth和screen.availHeight：可用宽度/高度，（即减去系统界面元素后的宽度和高度）
      • screen.colorDepth：颜色深度
      • screen.pixelDepth：颜色分辨率

  • 第二类：window窗口属性

    • 滚动属性
      • scrollY（或pageYOffset）：返回文档在垂直方向已滚动的像素值。
      • scrollTop：元素顶部到视口可见内容顶部的距离（只读），通常用于获取或设置特定元素的滚动位置（如<div>或<body>）。
      • scrollLeft：元素左边到视口可见内容左边的距离（只读），同样用于获取或设置特定元素的滚动位置。
    • 尺寸属性
      • innerWidth：返回浏览器窗口的视口（viewport）宽度（以像素为单位），包括右边距（如果有）。
      • innerHeight：返回浏览器窗口的视口（viewport）高度（以像素为单位），包括下边距（如果有）。
      • outerWidth：返回整个浏览器窗口的宽度（包括工具栏和滚动条）。
      • outerHeight：返回整个浏览器窗口的高度（包括工具栏和滚动条）。
    • 位置属性
      • screenX（或screenLeft）：返回窗口相对于屏幕的X坐标（只读）。
      • screenY（或screenTop）：返回窗口相对于屏幕的Y坐标（只读）。
    • 窗口控制属性

      • length：返回当前窗口中的iframe数量，<iframe> 即内联框架，常用于嵌入网页、广告、视频播放器、地图等

      • frames：返回当前窗口中的所有iframe的集合（类似于数组）

    • 其他属性
      • name：设置或返回窗口的名称。
      • status：设置或返回窗口的状态栏文本（尽管现代浏览器可能不再支持或显示它）。
      • history：返回History对象，它提供了与浏览器历史记录相关的功能。
      • document：返回对Document对象的引用，该对象表示窗口中加载的网页。
